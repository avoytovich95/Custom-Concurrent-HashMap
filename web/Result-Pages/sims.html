offline_bolt Simulations
Benchmark Explanation

This simulation is meant to show performace of the custom concurrent hashmap in an a micro application. The simulation runs a set of buyers/sellers which all "buy" and "sell". Essentialy, there are a set of threads that insert, delete, and modify the contents of a concurrent hashmap. The simulation is benchmarked against the built in benchmark with 5 warmup iterations, and 15 test iterations. The Wolf server had a higher compute copacity so it was given 32 Buyer/Seller instances, and the desktop was given 16.
